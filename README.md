# 🔐 **Secure_entry_system**



## 📘 **프로젝트 개요 (Overview)**

프로젝트는 STM32 마이크로컨트롤러 기반 출입 인증 제어 시스템으로,
사용자의 입력을 받아 인증을 수행하고 그 결과에 따라 출입 허용 또는 차단을 제어하는 임베디드 보안 시스템이다.

STM32CubeIDE 환경에서 GPIO, 인터럽트, 타이머 등 기본 제어 요소를 직접 구성하여
실제 하드웨어에서 안정적으로 동작하는 출입 제어 로직을 구현하는 것을 목표로 하였습니다.
시스템은 단순 제어를 넘어 상태 관리(FSM) 구조를 기반으로 확장 가능하도록 설계하였습니다.

---

## 🎯 **프로젝트 목표**

- STM32 기반 출입 인증 시스템의 전체 동작 흐름 설계 및 구현

- 사용자 입력 → 인증 → 제어 출력까지의 임베디드 제어 로직 이해 및 구현
- GPIO, 인터럽트, 타이머 등 MCU 핵심 주변장치 활용 능력 강화
- 인증 성공/실패에 따른 상태 분기 및 예외 처리 로직 구현

---

## ⚙️ **주요 기능**

### 🔹 출입 인증 입력처
- 카드  입력 신호 수신
- GPIO 입력 및 인터럽트 기반 이벤트 처리

### 🔹 인증 로직 수
- 사전 등록된 값과 입력 데이터 비교
- 인증 성공 / 실패 상태 판단

### 🔹 출입 제어 출력 
- 인증 성공 시 출입 허용 신호 출력
- 인증 실패 시 출입 차단 상태 유지

 ### 🔹 상태 표시 기능
- LED를 통한 시스템 상태 시각화
- 부저를 이용한 인증 성공/실패 알림

 ### 🔹 상태 관리(FSM) 구조
- 대기 → 입력 → 인증 → 허용/거부 단계별 상태 관리

## 🧠 ** 주요 구성 요소 **


| 구분    | 구성 요소            | 설명                          |
| ----- | ---------------- | --------------------------- |
| MCU   | STM32 마이크로컨트롤러   | 시스템 전체 제어, 입력·인증·출력 로직 수행   |
| 입력 장치 | 버튼 / 키패드 / 카드 입력 | 사용자 인증 정보 입력                |
| 입력 처리 | GPIO / 인터럽트      | 입력 이벤트를 실시간으로 감지 및 처리       |
| 인증 로직 | 펌웨어 로직 (C)       | 등록 정보 비교 및 인증 성공·실패 판단      |
| 출력 장치 | LED              | 시스템 상태(대기 / 성공 / 실패) 시각적 표시 |
| 출력 장치 | 부저 (Buzzer)      | 인증 결과에 따른 음향 알림             |
| 출입 제어 | 릴레이 / 전자락 / 서보모터 | 인증 성공 시 출입 허용 제어            |
| 상태 관리 | FSM 구조           | 단계별 상태 전이 및 예외 흐름 제어        |
| 개발 환경 | STM32CubeIDE     | 프로젝트 설정, 빌드 및 디버깅           |
| 전원    | USB / 외부 전원      | 시스템 구동을 위한 전원 공급            |


 

## 🚀 **배운 점**

이 프로젝트를 통해 STM32 기반으로 입력 처리, 인증 판단, 출력 제어까지의 전 과정을 직접 구현하며 
MCU 중심 임베디드 시스템의 전체 동작 흐름을 체계적으로 이해할 수 있었습니다.
출입 인증 제어 프로젝트는 임베디드 과정 중 첫 번째 프로젝트로, 수업에서 이론으로 접했을 때는 다소 추상적으로 느껴졌던 STM32 구조와 GPIO,
인터럽트, 타이머의 동작 원리를 실제 코드와 하드웨어 동작을 통해 경험하며 이해도를 크게 높일 수 있었습니다. 

또한 핵심 주변장치를 활용해 실제 하드웨어를 제어하면서 이론이 아닌 실전 중심의 임베디드 제어 감각을 익혔고, 
기능 확장을 고려한 펌웨어 구조를 설계함으로써 유지보수와 확장에 유리한 임베디드 소프트웨어 설계 경험을 쌓을 수 있었습니다.
